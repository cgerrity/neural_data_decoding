# Troubleshooting

Operational issues and their fixes. For *training* failures (NaN loss, OOM,
divergence, parity mismatches), see
[Debug a failing run](cookbook/debug_a_failing_run.md).

## Install and import

**`ModuleNotFoundError: neural_data_decoding`**
: The package isn't installed in the active environment. Activate the venv and
  `pip install -e ".[dev]"`.

**`pip install` fails on `matlabengine`**
: This project does **not** use the MATLAB Engine for Python (it can't bind a
  Rosetta Python to an ARM64 MATLAB). It uses the `matlab -batch` subprocess
  instead. If a stray `matlabengine` requirement appears, it's not needed.

## MATLAB interop

**"MATLAB not found"**
: Set `$MATLAB_EXECUTABLE`, add MATLAB to `$PATH`, or install it. For code that
  should tolerate its absence, guard calls with `matlab_available()`.

**A `needs_matlab` test is "skipped"**
: Expected without MATLAB — those tests are deselected by default
  (`-m "not needs_matlab"`). "Skipped" ≠ "failed."

**A MATLAB `-batch` call fails intermittently**
: Back-to-back MATLAB launches can contend on the license. Wrap the call in
  `try/except` and let it degrade gracefully.

## Paths and environment

**`FileNotFoundError` on the cluster but not locally**
: A hardcoded path bypassed `get_base_paths()`. Route every path through it.

**Detected the wrong environment**
: Force it: `NDD_FORCE_ENV=accre` (or `local`/`teba`), or
  `get_base_paths(environment=...)`.

**Results disappeared after a cluster job**
: They were written to scratch (`temporary`), which is purged. Write preserved
  output to `output`. See [Running on ACCRE](user_guide/running_on_accre.md).

## Config

**`FileNotFoundError` on `--config-name`**
: The name must match a file in `configs/target_milestone/`, minus `.yaml`.
  There are no Hydra config groups — `configs/architecture/` and `configs/sweep/`
  are empty.

**An `--override` did nothing**
: A misspelled key *adds* a new key instead of overriding. Confirm the key exists
  in the merged config.

## Notebooks

**A curriculum notebook won't execute**
: Ensure the dev environment is installed (`pip install -e ".[dev]"`, including
  `matplotlib`) and a kernel is selected. The notebooks are IDE-agnostic — see
  `notebooks/00_orientation/00.1_welcome.ipynb` §2 for VS Code vs JupyterLab
  kernel selection.

## Fixtures and parity

**A parity test errors on a missing fixture**
: Golden `.mat` fixtures are generated by `scripts/prepare_golden_fixtures.py`
  (needs MATLAB + the source root). The `reference_cm_tables` fixtures are small
  and committed; regenerate the golden weights if absent.
